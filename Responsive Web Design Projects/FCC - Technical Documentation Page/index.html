Introduction

JavaScript est un langage de script multiplateforme orienté objet. C'est un langage petit et léger. Dans un environnement hôte (par exemple, un navigateur Web), JavaScript peut être connecté aux objets de son environnement pour fournir un contrôle programmatique sur eux.

JavaScript contient une bibliothèque standard d'objets, tels que Array, Date et Math, et un ensemble principal d'éléments de langage tels que des opérateurs, des structures de contrôle et des instructions. Le JavaScript de base peut être étendu à diverses fins en le complétant avec des objets supplémentaires; par exemple :

  • JavaScript côté client étend le langage principal en fournissant des objets pour contrôler un navigateur et son modèle d'objet de document (DOM). Par exemple, les extensions côté client permettent à une application de placer des éléments sur un formulaire HTML et de répondre aux événements utilisateur tels que les clics de souris, la saisie de formulaire et la navigation dans les pages.
  • Le JavaScript côté serveur étend le langage principal en fournissant des objets pertinents pour exécuter JavaScript sur un serveur. Par exemple, les extensions côté serveur permettent à une application de communiquer avec une base de données, d'assurer la continuité des informations d'une invocation à une autre de l'application ou d'effectuer des manipulations de fichiers sur un serveur.

Qu'est ce que vous devriez savoir ?

Ce guide suppose que vous disposez des connaissances de base suivantes :

  • Une compréhension générale d'Internet et du World Wide Web (WWW).
  • Une bonne connaissance pratique du langage de balisage HyperText (HTML).
  • Une expérience en programmation. Si vous êtes nouveau dans la programmation, essayez l'un des tutoriels liés à la page principale sur JavaScript.

Javascript et Java

JavaScript et Java sont similaires à certains égards, mais fondamentalement différents à certains autres. Le langage JavaScript ressemble à Java mais n'a pas de typage statique ni de vérification de type forte de Java. JavaScript suit la plupart des syntaxes d'expressions Java, des conventions de dénomination et des constructions de contrôle de flux de base, raison pour laquelle il a été renommé de LiveScript en JavaScript.

Contrairement au système de classes de compilation de Java construit par des déclarations, JavaScript prend en charge un système d'exécution basé sur un petit nombre de types de données représentant des valeurs numériques, booléennes et de chaîne. JavaScript a un modèle objet basé sur un prototype au lieu du modèle objet basé sur une classe plus courant. Le modèle basé sur un prototype fournit un héritage dynamique; c'est-à-dire que ce qui est hérité peut varier pour des objets individuels. JavaScript prend également en charge des fonctions sans aucune exigence déclarative particulière. Les fonctions peuvent être des propriétés d'objets, s'exécutant comme des méthodes typées de manière lâche.

JavaScript est un langage de forme très libre par rapport à Java. Vous n'avez pas à déclarer toutes les variables, classes et méthodes. Vous n'avez pas à vous soucier de savoir si les méthodes sont publiques, privées ou protégées, et vous n'avez pas à implémenter d'interfaces. Les variables, les paramètres et les types de retour de fonction ne sont pas explicitement typés.

Hello World

Pour commencer à écrire JavaScript, ouvrez le Scratchpad et écrivez votre premier code JavaScript "Hello world" :

function greetMe(yourName) { alert("Hello " + yourName); }
greetMe("World");

Sélectionnez le code dans le pavé et appuyez sur Ctrl + R pour le voir se dérouler dans votre navigateur !

Les variables

Vous utilisez des variables comme noms symboliques pour les valeurs de votre application. Les noms des variables, appelés identifiants, sont conformes à certaines règles.

Un identifiant JavaScript doit commencer par une lettre, un trait de soulignement (_) ou un signe dollar ($); les caractères suivants peuvent également être des chiffres (0-9). Parce que JavaScript est sensible à la casse, les lettres incluent les caractères "A" à "Z" (majuscules) et les caractères "a" à "z" (minuscules).

Vous pouvez utiliser des lettres ISO 8859-1 ou Unicode telles que å et ü dans les identificateurs. Vous pouvez également utiliser les séquences d'échappement Unicode comme caractères dans les identificateurs. Quelques exemples de noms légaux sont Number_hits, temp99 et _name.

Déclarer des variables

Vous pouvez déclarer une variable de trois manières :

Avec le mot-clé var. Par exemple,

var x = 42.

Cette syntaxe peut être utilisée pour déclarer des variables locales et globales.

En lui attribuant simplement une valeur. Par exemple,

x = 42.

Cela déclare toujours une variable globale. Il génère un avertissement JavaScript strict. Vous ne devez pas utiliser cette variante.

Avec le mot-clé let. Par exemple,

let y = 13.

Portée des variables

Lorsque vous déclarez une variable en dehors de toute fonction, elle est appelée variable globale, car elle est disponible pour tout autre code dans le document actuel. Lorsque vous déclarez une variable dans une fonction, elle est appelée variable locale, car elle n'est disponible que dans cette fonction.

Lorsque vous déclarez une variable en dehors de toute fonction, elle est appelée variable globale, car elle est disponible pour tout autre code dans le document actuel. Lorsque vous déclarez une variable dans une fonction, elle est appelée variable locale, car elle n'est disponible que dans cette fonction.

if (true) { var x = 5; } console.log(x); // 5

Ce comportement change lors de l'utilisation de la déclaration let introduite dans ECMAScript 2015.

if (true) { let y = 5; } console.log(y); // ReferenceError: y is not defined

Les variables globales

Les variables globales sont en fait des propriétés de l'objet global. Dans les pages Web, l'objet global est window, vous pouvez donc définir et accéder aux variables globales à l'aide de la syntaxe window.variable.

Par conséquent, vous pouvez accéder aux variables globales déclarées dans une fenêtre ou un cadre à partir d'une autre fenêtre ou d'un autre cadre en spécifiant le nom de la fenêtre ou du cadre. Par exemple, si une variable appelée phoneNumber est déclarée dans un document, vous pouvez faire référence à cette variable à partir d'un iframe en tant que parent.phoneNumber.

Les constantes

Vous pouvez créer une constante nommée en lecture seule avec le mot clé const. La syntaxe d'un identificateur de constante est la même que celle d'un identificateur de variable: elle doit commencer par une lettre, un trait de soulignement ou un signe dollar et peut contenir des caractères alphabétiques, numériques ou de soulignement.

const PI = 3.14;

Une constante ne peut pas changer de valeur par affectation ni être déclarée de nouveau pendant l'exécution du script. Il doit être initialisé à une valeur.

Les règles d'étendue des constantes sont les mêmes que celles des variables d'étendue de bloc let. Si le mot clé const est omis, l'identifiant est supposé représenter une variable.

Vous ne pouvez pas déclarer une constante portant le même nom qu'une fonction ou variable dans la même portée. Par exemple :

// THIS WILL CAUSE AN ERROR function f() {}; const f = 5;
// THIS WILL CAUSE AN ERROR ALSO function f() { const g = 5; var g; //statements }

Cependant, les attributs d'objet ne sont pas protégés, donc l'instruction suivante est exécutée sans problème.

const MY_OBJECT = {"key": "value"}; MY_OBJECT.key = "otherValue";

Les types de données

La dernière norme ECMAScript définit sept types de données :

  • Six types de données primitifs :
    • Booléen. vrai et faux.
    • null. Un mot-clé spécial indiquant une valeur nulle. Parce que JavaScript est sensible à la casse, null n'est pas le même que Null, NULL ou toute autre variante.
    • indéfini. Une propriété de niveau supérieur dont la valeur n'est pas définie.
    • Nombre. 42 ou 3.14159.
    • Chaîne. "Salut"
    • Symbole (nouveau dans ECMAScript 2015). Un type de données dont les instances sont uniques et immuables.
  • et les objets

Bien que ces types de données soient relativement peu nombreux, ils vous permettent d'effectuer des fonctions utiles avec vos applications. Les objets et les fonctions sont les autres éléments fondamentaux du langage. Vous pouvez considérer les objets comme des conteneurs nommés pour les valeurs et les fonctions comme des procédures que votre application peut exécuter.

If...Else

Utilisez l'instruction if pour exécuter une instruction si une condition logique est vraie. Utilisez la clause else facultative pour exécuter une instruction si la condition est fausse. Une instruction if se présente comme suit:

if (condition) { statement_1; } else { statement_2; }

Une condition peut être n'importe quelle expression évaluée à vrai ou faux. Voir Boolean pour une explication de ce qui vaut vrai et faux. Si la condition prend la valeur true, l'instruction_1 est exécutée; sinon, statement_2 est exécuté. instruction_1 et instruction_2 peuvent être n'importe quelle instruction, y compris les instructions if imbriquées supplémentaires.

Vous pouvez également composer les instructions en utilisant else if pour que plusieurs conditions soient testées en séquence, comme suit :

if (condition_1) { statement_1; }
else if (condition_2) { statement_2;}
else if (condition_n) { statement_n; }
else { statement_last; }

Dans le cas de conditions multiples, seule la première condition logique évaluée à vrai sera exécutée. Pour exécuter plusieurs instructions, regroupez-les dans une instruction de bloc ({...}). En général, il est recommandé de toujours utiliser des instructions de bloc, en particulier lors de l'imbrication d'instructions if :

if (condition) { statement_1_runs_if_condition_is_true;
statement_2_runs_if_condition_is_true; } else {
statement_3_runs_if_condition_is_false;
statement_4_runs_if_condition_is_false; }

Il est conseillé de ne pas utiliser d'affectations simples dans une expression conditionnelle, car l'affectation peut être confondue avec l'égalité lors de la consultation du code. Par exemple, n'utilisez pas le code suivant:

if (x = y) { /* statements here */ }

Si vous devez utiliser une affectation dans une expression conditionnelle, une pratique courante consiste à mettre des parenthèses supplémentaires autour de l'affectation. Par exemple :

if ((x = y)) { /* statements here */ }

While

Une instruction while exécute ses instructions tant qu'une condition spécifiée a la valeur true. Une instruction while se présente comme suit :

while (condition) statement

Si la condition devient fausse, l'instruction dans la boucle cesse de s'exécuter et le contrôle passe à l'instruction qui suit la boucle.

Le test de condition se produit avant l'exécution de l'instruction dans la boucle. Si la condition renvoie true, l'instruction est exécutée et la condition est testée à nouveau. Si la condition retourne false, l'exécution s'arrête et le contrôle est passé à l'instruction suivante while.

Pour exécuter plusieurs instructions, utilisez une instruction de bloc ({...}) pour regrouper ces instructions.

Exemple:

La boucle while suivante itère tant que n est inférieur à trois:

var n = 0; var x = 0; while (n > 3) { n++; x += n; }

À chaque itération, la boucle incrémente n et ajoute cette valeur à x. Par conséquent, x et n prennent les valeurs suivantes:

  • Après le premier passage: n = 1 et x = 1
  • Après le deuxième passage: n = 2 et x = 3
  • Après le troisième passage: n = 3 et x = 6

Après avoir terminé la troisième passe, la condition n > 3 n'est plus vraie, donc la boucle se termine.

Déclarer une fonction

Une définition de fonction (également appelée déclaration de fonction ou instruction de fonction) se compose du mot clé function, suivi de :

  • Le nom de la fonction.
  • Une liste d'arguments de la fonction, entre parenthèses et séparés par des virgules.
  • Les instructions JavaScript qui définissent la fonction, entre crochets, {}.

Par exemple, le code suivant définit une fonction simple nommée carré :

function square(number) { return number * number; }

La fonction carré prend un argument, appelé nombre. La fonction se compose d'une instruction qui dit de renvoyer l'argument de la fonction (c'est-à-dire, nombre) multiplié par lui-même. L'instruction return spécifie la valeur renvoyée par la fonction.

return number * number;

Les paramètres primitifs (comme un nombre) sont passés aux fonctions par valeur; la valeur est transmise à la fonction, mais si la fonction modifie la valeur du paramètre, cette modification n'est pas reflétée globalement ou dans la fonction appelante.
